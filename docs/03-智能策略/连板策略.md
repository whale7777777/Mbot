# 连板策略

基于涨停池数据的**连板晋级概率**统计、滚动回测校准与每日选股参考。适用于 A 股短线情绪周期中，对「昨日连板股今日能否再多一封」进行量化筛选（非自动下单，需结合盘面与风控自行决策）。

---

## 1. 策略逻辑

### 1.1 核心概念

| 术语 | 定义 |
|------|------|
| **n 连板** | 当日收盘涨停池中，该股「连板数」字段为 n（首板 n=1） |
| **晋级** | T 日 n 连板的个股，T+1 日仍在涨停池且连板数为 **n+1** |
| **晋级概率** | 历史上同连板高度 n 的样本中，完成晋级的比例（经平滑与因子修正） |
| **连板股（本策略默认）** | 连板数 ≥ 2，剔除 ST、*ST 及名称含「退」的标的 |

媒体常说的「连板晋级率」一般指：**T 日连板数 ≥ 2** 的样本在 T+1 再多封一天的比例（不含首板）。

### 1.2 预测思路

1. 用 **T 日之前** 若干相邻交易日的涨停池，统计各连板高度 n → n+1 的实证晋级率。
2. 对样本少的档位做 **拉普拉斯平滑**，并向整体连板晋级率 **收缩**，避免极端估计。
3. 结合个股当日 **封板质量因子**（炸板次数、封板资金等）做乘子修正。
4. 用近段交易日做 **滚动回测**，按分层偏差微调各档概率，写入校准文件供次日使用。

### 1.3 因子说明（校准后）

| 因子键 | 含义 | 规则 |
|--------|------|------|
| `zero_zhaban` | 无炸板 | 当日炸板次数 = 0 |
| `heavy_seal` | 强封单 | 封板资金 ≥ 1 亿元 |
| `early_seal` | 早盘封板 | 首次封板时间不晚于 10:30 |

因子以**相对基准晋级率的乘子**形式叠加（回测学习得到，见校准 JSON）。极高连板（样本极少）时，回退到不高于当前高度的最近一档历史率。

## 2. 数据与配置

### 2.1 数据源

| 优先级 | 来源 | 接口 | 历史长度 |

```json

  "tushare_token": "",
  "tushare_token_env": "TUSHARE_TOKEN",
  "read_token_from": "../../utils/configure/config.json",
  "read_token_field": "ts_token",
  "fallback_to_em": true,
  "trade_days_target": 35,

  "validate_pairs": 30,

  "laplace_alpha": 1.0,
  "shrink_to_overall": 0.15

```

**Token 优先级（高 → 低）**

1. 环境变量 `LIANBAN_TUSHARE_TOKEN`
2. 环境变量 `TUSHARE_TOKEN`
3. `lianban_config.json` → `tushare_token`
4. `utils/configure/config.json` → `ts_token`

TuShare 涨跌停列表需 [Pro 积分](https://tushare.pro/document/2?doc_id=298)。

### 3.2 推荐使用流程

```bash
# ① 首次：生成配置（若尚无 lianban_config.json）
python scripts/lianban_data.py --init

# ② 配置 Token 后检查
python scripts/lianban_data.py --test

# ③ 滚动回测并更新校准（建议每周或行情阶段切换后重跑）
python scripts/lianban_backtest_30d.py --days 35 --lookback 20

# ④ 收盘后生成当日连板分析
python scripts/lianban_today.py

# 可选：指定交易日
python scripts/lianban_today.py --date 20260519

# 可选：含首板
python scripts/lianban_today.py --min-boards 1
```

## 4. 晋级概率计算公式（简述）

对连板数为 \(n\) 的个股：

1. **基准率** \(p_n\)：由 T 日前 `lookback_pairs` 对相邻日中，所有「T 日 n 连板 → T+1 日 n+1」的样本汇总，再经拉普拉斯平滑与向整体率收缩。
2. **因子修正**：\(p = \mathrm{clip}\big(p_n \times \prod_i m_i,\ 0.02,\ 0.92\big)\)，其中 \(m_i\) 为触发的因子乘子（如 `zero_zhaban`、`heavy_seal`）。
3. **分层微调**：滚动回测后对各档 \(n\) 做有限幅度（约 ±15%）修正，写入 `连板预测校准.json`。

**命中判定（回测）**：T 日收盘后给出预测；T+1 收盘后若该股仍在涨停池且连板数 = n+1，则记为晋级命中。

## 5. 回测指标说明

| 指标 | 含义 |
|------|------|
| **Brier 分数** | 概率预测均方误差，越低越好 |
| **阈值 0.5 命中率** | 以 50% 为界二分类的准确率（仅供参考） |
| **实证晋级率** | 验证样本中实际晋级的比例 |

回测采用**严格 walk-forward**：预测 T 日个股时，仅使用 T 日之前的交易对，不使用未来信息。

未配置 TuShare 时，历史仅约 15 日，脚本会自动降低 `lookback` 与验证对数，并在 stderr 提示配置 token 以跑满 30 日。

## 6. 使用建议与风险

1. **概率≠建议满仓**：晋级概率用于排序与过滤，需结合板块强度、龙头地位、监管与流动性。
2. **定期重校准**：情绪周期切换后应重跑 `lianban_backtest_30d.py`。
3. **数据延迟**：TuShare 日线涨停数据通常在收盘后更新；盘中请以实时行情为准。
4. **样本偏差**：极高连板（6 板以上）样本少，估计不稳定，宜降仓或观望。
5. **合规**：本工具仅供研究学习，不构成投资建议。

## 8. 常见问题

**Q：晋级概率全是 0 或异常？**  
检查相邻交易日顺序是否正确；需先跑回测生成校准文件，并确认数据源能拉到 T 与 T+1 两日涨停池。

**Q：只有 15 个交易日？**  
当前回退到了东方财富接口。请配置 TuShare token 并将 `data_source` 设为 `tushare`。

**Q：与「涨停开板」策略的关系？**  
连板策略关注**连续封板能否延续**；开板策略关注涨停后打开再封等盘口行为，二者可并列使用、分别建模。
